# Personal Website

This is my personal website project, which is a fork of [another repository](https://github.com/barryclark/jekyll-now).

## Overview

This project uses **Jekyll**, a static site generator that's perfect for GitHub-hosted blogs. **Jekyll Now** simplifies the setup process, allowing you to create a Jekyll blog without needing to touch the command line or install/configure Ruby and its dependencies.

## Features

- No command-line usage required
- No need to install/configure Ruby, rvm/rbenv, or Ruby gems
- No need to install runtime dependencies like markdown processors or Pygments
- Easy setup for Windows users
- Minimal, responsive blog setup in minutes

## Quick Start

### Step 1: Fork the Repository

Fork this repository and rename it to `yourgithubusername.github.io`. Your Jekyll blog will be viewable at `https://yourgithubusername.github.io`.

### Step 2: Customize Your Site

Edit the `_config.yml` file to enter your site name, description, avatar, and other options. This will force GitHub Pages to rebuild your site with Jekyll.

### Step 3: Publish Your First Blog Post

Edit `/_posts/2014-3-3-Hello-World.md` to publish your first blog post. Refer to the [Markdown Cheatsheet](http://www.jekyllnow.com/Markdown-Style-Guide/) for help with Markdown syntax.

## Local Development

1. Install Jekyll and plugins: `gem install github-pages`
2. Clone your fork: `git clone https://github.com/yourusername/yourusername.github.io.git`
3. Serve the site: `jekyll serve`
4. View your site at `http://127.0.0.1:4000/`
5. Commit and push changes to the master branch of your GitHub repository.
